2 In this section, we will describe a number of examples from the
3 *example/embedded* folder. To allow code-reuse with different platforms
4 as well as with new ports, the low-level initialization of BTstack and
5 the hardware configuration has been extracted to the various
6 *platforms/PLATFORM/main.c* files. The examples only contain the
7 platform-independent Bluetooth logic. But let’s have a look at the
8 common init code.
9 
10 Listing [below](#lst:btstackInit) shows a minimal platform setup for an
11 embedded system with a Bluetooth chipset connected via UART.
12 
13 ~~~~ {#lst:btstackInit .c caption="{Minimal platform setup for an embedded system}"}
14 
15  int main(){
16  // ... hardware init: watchdoch, IOs, timers, etc...
17 
18  // setup BTstack memory pools
19  btstack_memory_init();
20 
21  // select embedded run loop
22  run_loop_init(RUN_LOOP_EMBEDDED);
23 
24  // use logger: format HCI_DUMP_PACKETLOGGER, HCI_DUMP_BLUEZ or HCI_DUMP_STDOUT
25  hci_dump_open(NULL, HCI_DUMP_STDOUT);
26 
27  // init HCI
28  hci_transport_t * transport = hci_transport_h4_dma_instance();
29  remote_device_db_t * remote_db = (remote_device_db_t *) &remote_device_db_memory;
30  hci_init(transport, NULL, NULL, remote_db);
31 
32  // setup example
33  btstack_main(argc, argv);
34 
35  // go
36  run_loop_execute();
37  }
38 
39 ~~~~
40 
41 First, BTstack’s memory pools are setup up. Then, the standard run loop
42 implementation for embedded systems is selected.
43 
44 The call to *hci_dump_open* configures BTstack to output all Bluetooth
45 packets and it’s own debug and error message via printf. The Python
46 script *tools/create_packet_log.py* can be used to convert the console
47 output into a Bluetooth PacketLogger format that can be opened by the OS
48 X PacketLogger tool as well as by Wireshark for further inspection. When
49 asking for help, please always include a log created with HCI dump.
50 
51 The *hci_init* function sets up HCI to use the HCI H4 Transport
52 implementation. It doesn’t provide a special transport configuration nor
53 a special implementation for a particular Bluetooth chipset. It makes
54 use of the *remote_device_db_memory* implementation that allows for
55 re-connects without a new pairing but doesn’t persist the bonding
56 information.
57 
58 Finally, it calls *btstack_main()* of the actual example before
59 executing the run loop.
