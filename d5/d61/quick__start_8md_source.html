2 ## General Tools
3 
4 On Unix-based systems, git, make, and Python are usually installed. If
5 not, use the system’s packet manager to install them.
6 
7 On Windows, you need to manually install and configure GNU Make, Python,
8 and optionally git :
9 
10 - [GNU Make](http://gnuwin32.sourceforge.net/packages/make.htm)
11  for Windows: Add its bin folder to the Windows Path in Environment
12  Variables. The bin folder is where make.exe resides, and it’s
13  usually located in [C:\Program Files\GnuWin32\bin]().
14 
15 - [Python](http://www.python.org/getit/) for
16  Windows: Add Python installation folder to the Windows Path in
17  Environment Variables.
18 
19 ### Adding paths to the Windows Path variable {#sec:windowsPathQuickStart}
20 
21 - Go to: Control Panel->System->Advanced tab->Environment Variables.
22 
23 - The top part contains a list of User variables.
24 
25 - Click on the Path variable and then click edit.
26 
27 - Go to the end of the line, then append the path to the list, for
28  example, [C:\Program Files\GnuWin32\bin]() for GNU Make.
29 
30 - Ensure that there is a semicolon before and after [C:\Program Files\GnuWin32\bin]().

43 ## Compiling the examples and loading firmware
44 
45 This step is platform specific. To compile and run the examples, you
46 need to download and install the platform specific toolchain and a flash
47 tool. For TI’s CC256x chipsets, you also need the correct init script,
48 or “Service Pack” in TI nomenclature. Assuming that these are provided,
49 go to folder [btstack/platforms/$PLATFORM$]() in command prompt and run make.
50 If all the paths are correct, it will generate several firmware files. These firmware files
51 can be loaded onto the device using platform specific flash programmer.
52 For the PIC32-Harmony platform, a project file for the MPLAB X IDE is
53 provided, too.
54 
55 ## Run the Example
56 
57 As a first test, we recommend the [SPP Counter example](examples/generated/#sec:sppcounterExample).
58 During the startup, for TI chipsets, the init
59 script is transferred, and the Bluetooth stack brought up. After that,
60 the development board is discoverable as “BTstack SPP Counter” and
61 provides a single virtual serial port. When you connect to it, you’ll
62 receive a counter value as text every second.

69 ### libusb
70 
71 The quickest way to try BTstack is on a Linux or OS X system with an
72 additional USB Bluetooth module. The Makefile [platforms/libusb]() in requires
73 [pkg-config](http://www.freedesktop.org/wiki/Software/pkg-config/)
74 and [libusb-1.0](http://libusb.info) or higher to be
75 installed.
76 
77 On Linux, it’s usually necessary to run the examples as root as the
78 kernel needs to detach from the USB module.
79 
80 On OS X, it’s necessary to tell the OS to only use the internal
81 Bluetooth. For this, execute:
82 
83  sudo nvram bluetoothHostControllerSwitchBehavior=never
84 
85 It’s also possible to run the examples on Win32 systems. For this:
86 
87 - Install [MSYS](http://www.mingw.org/wiki/msys) and
88  [MINGW32](http://www.mingw.org) using the MINGW installer
89 
90 - Compile and install libusb-1.0.19 to [/usr/local/]() in msys command
91  shell
92 
93 - Setup a USB Bluetooth dongle for use with libusb-1.0:
94 
95  - Start [Zadig](http://zadig.akeo.ie)
96 
97  - Select Options -> “List all devices”
98 
99  - Select USB Bluetooth dongle in the big pull down list
100 
101  - Select WinUSB (libusb) in the right pull pull down list
102 
103  - Select “Replace Driver”
104 
105 Now, you can run the examples from the *msys* shell the same way as on
106 Linux/OS X.

108 ### Texas Instruments MSP430-based boards
109 
110 **Compiler Setup.** The MSP430 port of BTstack is developed using the
111 Long Term Support (LTS) version of mspgcc. General information about it
112 and installation instructions are provided on the
113 [MSPGCC Wiki](http://sourceforge.net/apps/mediawiki/mspgcc/index.php?title=MSPGCC_Wiki).
114 On Windows, you need to download and extract
115 [mspgcc](http://sourceforge.net/projects/mspgcc/files/Windows/mingw32/)
116 to [C:\mspgcc](). Add [C:\mspgcc\bin]() folder to the Windows Path in Environment
117 variable as explained [here](#sec:windowsPathQuickStart).
118 
119 **Loading Firmware.** To load firmware files onto the MSP430 MCU for the
120 MSP-EXP430F5438 Experimeneter board, you need a programmer like the
121 MSP430 MSP-FET430UIF debugger or something similar. The eZ430-RF2560 and
122 MSP430F5529LP contain a basic debugger. Now, you can use one of
123 following software tools:
124 
125 - [MSP430Flasher](http://processors.wiki.ti.com/index.php/MSP430_Flasher_Command_Line_Programmer)
126  (windows-only):
127 
128  - Use the following command, where you need to replace the [BINARY_FILE_NAME.hex]() with
129  the name of your application:
130 
131 <!-- -->
132 
133  MSP430Flasher.exe -n MSP430F5438A -w "BINARY_FILE_NAME.hex" -v -g -z [VCC]
134 
135 
136 - [MSPDebug](http://mspdebug.sourceforge.net/):
137  An example session with the MSP-FET430UIF connected on OS X is given
138  in following listing:
139 
140 <!-- -->
141 
142  mspdebug -j -d /dev/tty.FET430UIFfd130 uif
143  ...
144  prog blink.hex
145  run

147 ### Texas Instruments CC256x-based chipsets
148 
149 **CC256x Init Scripts.** In order to use the CC256x chipset on the
150 PAN13xx modules and others, an initialization script must be obtained.
151 Due to licensing restrictions, this initialization script must be
152 obtained separately as follows:
153 
154 - Download the [BTS file](http://processors.wiki.ti.com/index.php/CC256x_Downloads)
155  for your CC256x-based module.
156 
157 - Copy the included .bts file into
158 
159 - In [chipset-cc256x](), run the Python script:
160 
161 <!-- -->
162 
163  ./convert_bts_init_scripts.py
164 
165 The common code for all CC256x chipsets is provided by
166 *bt_control_cc256x.c*. During the setup,
167 *bt_control_cc256x_instance* function is used to get a
168 *bt_control_t* instance and passed to *hci_init* function.
169 
170 **Note:** Depending on the CC256x-based module you’re using, you’ll need
171 to update the reference in the Makefile to match the downloaded file.
172 
173 **Update:** For the latest revision of the CC256x chipsets, the CC2560B
174 and CC2564B, TI decided to split the init script into a main part and
175 the BLE part. The conversion script has been updated to detect
176 *bluetooth_init_cc256x_1.2.bts* and adds *BLE_init_cc256x_1.2.bts*
177 if present and merges them into a single .c file.
178 
179 **Update 2:** In May 2015, TI renamed the init scripts to match
180 the naming scheme previously used on Linux systems. The conversion
181 script has been updated to also detect *initscripts_TIInit_6.7.16_bt_spec_4.1.bts*
182 and integrates *initscripts_TIInit_6.7.16_ble_add-on.bts* if present.

196 ### PIC32 Bluetooth Audio Development Kit
197 
198 The PIC32 Bluetooth Audio Development Kit comes with the CSR8811-based
199 BTM805 Bluetooth module. In the port, the UART on the DAC daughter board
200 was used for the debug output. Please remove the DAC board and connect a
201 3.3V USB-2-UART converter to GND and TX to get the debug output.
202 
203 In [platforms/pic32-harmony](), a project file for the MPLAB X IDE
204 is provided as well as a regular
205 Makefile. Both assume that the MPLAB XC32 compiler is installed. The
206 project is set to use -Os optimization which will cause warnings if you
207 only have the Free version. It will still compile a working example. For
208 this platform, we only provide the SPP and LE Counter example directly.
209 Other examples can be run by replacing the *spp_and_le_counter.c* file
210 with one of the other example files.
